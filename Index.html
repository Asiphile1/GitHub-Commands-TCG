git clone

Creates a local copy of a remote repository.

git clone <repo-url>
git clone https://github.com/user/repo.git

This command downloads all files, history, and branches from a remote repository to your local machine for development.

git push

Uploads local commits to a remote repository.

git push <remote> <branch>
git push origin main

Sends your committed changes to a remote repository, making them available to collaborators.

Back to Command List

git pull

Fetches and merges changes from a remote repository into your current branch.

git pull <remote> <branch>
git pull origin main

Updates your local branch with the latest changes from the remote, combining fetch and merge in one step.

git commit

Records changes to the repository with a message.

git commit -m "message"
git commit -m "Add feature"

Saves your staged changes as a new snapshot in the project history, allowing you to track progress and roll back if needed.

git add

Adds changes in the working directory to the staging area, preparing them to be committed.

git add [file]
git add README.md

This command stages the specified file(s) so that they will be included in the next commit. You can also add multiple files or use git add . to stage all changes in the directory.

Back to Command List

git status

Displays the state of the working directory and staging area.

git status
git status

This command shows which changes have been staged, which haven’t, and which files aren’t being tracked by Git. It helps you understand what will be included in the next commit and what still needs attention.

git branch

Lists, creates, or deletes branches in your repository.

git branch
  git branch [branch-name]
  git branch -d [branch-name]
git branch feature-login
  git branch -d feature-login

Without arguments, git branch lists all local branches. You can create a new branch by specifying its name, or delete a branch with the -d flag. Branches allow you to work on different features or fixes independently.

Back to Command List

git checkout

Switches between branches or restores files in the working directory.

git checkout [branch-name]
  git checkout -b [new-branch-name]
git checkout main
  git checkout -b feature-ui

This command lets you move between branches in your repository. Using -b creates a new branch and switches to it immediately, enabling streamlined feature development or bug fixes.
